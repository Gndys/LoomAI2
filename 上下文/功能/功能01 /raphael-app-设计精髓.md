# Raphael.app 设计精髓分析

## 一、核心交互设计

### 1. 参数控制的极简设计

**不是这样（传统方式）**：
```
强度: [━━━●━━━] 0.4
背景: ○白色 ○灰色 ○透明
尺寸: ○1:1 ○16:9 ○9:16
```

**而是这样（raphael.app 方式）**：
```
□ Auto  □ 无风格  □ 精确度  □ 其他
```
- 点击才弹出选项菜单
- 默认状态极简
- 高级用户才展开

---

### 2. 提示词编辑器的渐进式设计

**基础状态**：
```
提示词: [你想看到什么？]
```

**点击后展开**：
```
提示词: [你想看到什么？]

模型: Loom Pro  ○ 正面提示词  ○ 负面提示词

[清除]  [随机]  [生成 🚀]
```

**关键点**：
- 负面提示词默认隐藏
- 点击"负面提示词"才展开输入框
- 随机按钮降低使用门槛

---

### 3. 功能选择的横向滚动设计

**不是网格布局，而是横向滚动**：
```
← [📸平铺图] [✏️线稿] [🎨宣传图] [📐版型] [🔄改色] →
   [预览图]   [预览图]  [预览图]   [预览图]  [预览图]
   商品展示   技术制版  社交媒体   结构分析  配色方案
```

**优势**：
- 节省垂直空间
- 可以放更多功能
- 视觉更流畅

---

### 4. 模型命名的品牌化

**不要这样**：
```
模型: Gemini 2.0 Flash
```

**要这样**：
```
模型: Loom Pro
```

**原因**：
- 用户不关心底层技术
- 品牌化命名更专业
- 建立产品认知

---

### 5. 案例展示的瀑布流布局

**不是轮播图，而是瀑布流**：
```
[所有模型] [大师模型] [Loom Pro] [精选效果] [实时特效]

┌──────┐  ┌──────┐  ┌──────┐
│案例1 │  │案例2 │  │案例3 │
│ 线稿 │  │ 艺术 │  │      │
└──────┘  └──────┘  └──────┘

┌──────┐  ┌──────┐  ┌──────┐
│案例4 │  │案例5 │  │案例6 │
│      │  │      │  │      │
└──────┘  └──────┘  └──────┘
```

**优势**：
- 一次展示更多案例
- 用户可以自由浏览
- 建立信任感

## 二、页面结构

### 完整的用户旅程

```
1. 首屏（Hero）
   ↓
   产品介绍 + 立即试用按钮
   
2. AI Image Generator（核心功能区）
   ↓

3. Try a style（功能选择）
   ↓

4. Get Inspired（案例展示）
   ↓
   瀑布流布局，建立信任
   
5. 产品特点说明
   ↓
   6个特点，图标 + 文字
   
6. 常见问题解答
   ↓
   FAQ
   
7. 页脚
   ↓
   链接导航
```

## 三、关键设计原则

### 1. 渐进式披露（Progressive Disclosure）

**不要一次性展示所有参数**，而是：
- 默认状态：极简（只有必要的）
- 点击展开：高级选项
- 用户自主选择深度

### 2. 品牌化命名

**技术术语 → 产品术语**：
- Gemini → Loom Pro
- Image-to-Image → AI 照片编辑器
- Negative Prompt → 负面提示词（可选）

### 3. 降低使用门槛

**提供快捷方式**：
- 随机按钮（不知道写什么？点随机）
- 风格预设（不知道选什么？点预设）
- 清除按钮（想重来？一键清空）

### 4. 视觉引导

**用颜色和大小引导用户**：
- 主要操作：橙色大按钮（生成）
- 次要操作：灰色小按钮（清除、随机）
- 危险操作：红色（删除）

## 四、我们要学习的核心点

### 1. 参数控制改成下拉菜单
```javascript
// 不要用滑块
<input type="range" />

// 要用下拉菜单
<button onClick={() => setShowMenu(true)}>
  Auto ▼
</button>
```

### 2. 负面提示词默认隐藏
```javascript
const [showNegative, setShowNegative] = useState(false)

<button onClick={() => setShowNegative(true)}>
  + 负面提示词
</button>

{showNegative && <textarea />}
```

### 3. 功能选择用横向滚动
```css
.function-list {
  display: flex;
  overflow-x: auto;
  scroll-snap-type: x mandatory;
}
```

### 4. 模型命名品牌化
```javascript
// 不要暴露技术细节
model: "gemini-2.0-flash-exp"

// 要用产品名称
model: "Loom Pro"
```

### 5. 添加随机按钮
```javascript
<button onClick={generateRandomPrompt}>
  🎲 随机提示词
</button>
```

## 七、总结

Raphael.app 的设计精髓在于：
1. **极简主义**：默认状态只显示必要的
2. **渐进式**：用户自主选择深度
3. **品牌化**：技术术语产品化
4. **降低门槛**：提供快捷方式（随机、预设）
5. **视觉引导**：用颜色和大小引导操作

我们要做的就是：**完全复刻这套交互逻辑，但内容换成服装行业的**。

# 模特氛围图功能页 PRD（V1）

> 版本：V1.0  
> 日期：2026-02-07  
> 基于会议：2026-02-05 方向讨论（工具化冷启动，强调“全链路出图”）

## 1. 背景与目标

### 1.1 背景
团队当前明确采用“垂直工具冷启动”路径：先做服装商家高频、可直接成交的素材能力，而非先做泛创意平台。

在既有白底图能力基础上，模特氛围图是最自然的下一跳：
- 白底图解决“可上架”
- 模特氛围图解决“看起来更想买”

一句话：
**把商品图在 3 分钟内转成可投放、可发圈、可上新的模特氛围图。**

### 1.2 目标（对标策略）
不和通用 AI 绘图比“自由创作”，而是比“服装电商转化素材效率”。

V1 目标是让小老板看到“直接能用、立即省钱、省沟通成本”的结果。

### 1.3 V1 业务目标（2周）
- 试用商家数：15–20 家（与白底图可重叠）
- 首次出图成功率：>= 75%（一次可用）
- 7日复用率：>= 35%
- 白底图到模特图转化率：>= 40%

---

## 2. 用户与场景

### 2.1 核心用户画像
- 服装档口老板：要“快出一批可发图”
- 小品牌运营：要“统一调性 + 批量稳定”
- 微商/私域操盘手：要“朋友圈素材持续更新”

### 2.2 核心任务（JTBD）
- 当我要发新款时，我希望不拍新片也能快速拿到有氛围、有模特感的素材。
- 当我要多渠道分发时，我希望同一款能批量产出不同构图比例和风格版本。

### 2.3 典型触发场景
- 每日/每周上新
- 活动档期前素材补量
- 门店或私域需要“今日可发”内容

---

## 3. 方案边界

### 3.1 V1 必做（MVP）
1. 支持上传商品图（优先接白底图）
2. 模板化风格生成（至少 4 个模板）
3. 基础模特设定（性别/年龄段/体型/姿态）
4. 批量生成与批量导出
5. 结果对比与一键重试
6. 基础质检提示（服装完整度/人像异常/文字噪点）

### 3.2 V1 可做（若工期允许）
1. 同款多场景一键出 3 版
2. 白底图结果页一键跳转并自动带图
3. 模板收藏（商家常用）

### 3.3 V1 不做（明确排除）
- 自由复杂 Prompt 工作台
- 局部重绘/骨骼级编辑
- 视频生成
- 全行业泛化优化（先聚焦服装）

---

## 4. 信息架构与流程

### 4.1 页面主流程
1. 选择输入图（上传或从白底图流转）
2. 选择模板（通勤/门店/街拍/影棚）
3. 选择模特参数（默认推荐，可微调）
4. 执行生成（单张或批量）
5. 结果预览与质检
6. 批量导出
7. 流转到海报能力

### 4.2 页面区块
- 顶部价值区：价值承诺 + 示例前后对比
- 输入区：拖拽上传 + 批量队列
- 模板区：模板卡片 + 示例图
- 模特设定区：快速选项
- 结果区：网格预览 + 原图对比
- 质检区：异常提醒与建议
- 导出区：尺寸/命名/格式
- 下一步区：去生成海报

---

## 5. 详细功能需求（字段级）

### 5.1 输入区
字段：
- 上传方式：点击上传、拖拽上传、白底图一键导入
- 格式支持：JPG、PNG、WEBP
- 单张大小：<= 15MB
- 批量上限：V1 建议 <= 20 张

显示字段：
- 文件名、分辨率、状态、失败原因

校验规则：
- 分辨率过低提示“结果可能模糊”
- 检测到非服装主体时提示“建议上传单品图”

### 5.2 模板区
V1 模板建议（默认推荐顺序）：
1. 日常通勤感
2. 轻奢门店感
3. 街拍高级感
4. 电商影棚感

模板字段：
- 模板名称
- 场景描述（简短）
- 适用渠道（朋友圈/小红书/电商详情）
- 示例图（前后对比）

交互原则：
- 新手点击模板即可生成
- 高级用户可在“高级设置”中微调

### 5.3 模特设定区
字段：
- 性别：女 / 男 / 中性
- 年龄段：18-24 / 25-30 / 30+
- 体型：纤细 / 标准 / 偏丰满
- 姿态：站姿 / 走动 / 半身特写
- 表情：自然微笑 / 冷感 / 不露脸

交互规则：
- 默认开启“智能推荐”
- 修改任意字段后，显示“已自定义”标签

### 5.4 构图与输出设置区
字段：
- 构图：全身 / 半身 / 近景细节
- 比例：1:1 / 4:5 / 9:16
- 背景强度：低 / 中 / 高
- 品牌元素保护：开启（默认）

说明：
- 保证服装主体不被背景抢焦点
- 避免强背景导致商品细节被吞没

### 5.5 结果区
- 展示：网格卡片 + 单图详情
- 对比：原图/结果图滑块
- 版本：每图最多保留 3 个版本
- 操作：下载、再生成、复制同参数

状态：
- 处理中
- 成功
- 建议复检
- 失败

### 5.6 质检区
自动检测项：
- 服装完整度（是否缺失袖口/下摆）
- 人像自然度（手部、肢体、五官异常）
- 文字/Logo 污染（背景脏字）
- 商品可辨识度（细节是否过度平滑）

输出等级：
- 通过
- 警告（可导出）
- 建议重试（附推荐动作）

### 5.7 导出区
字段：
- 格式：JPG / PNG
- 质量：标准 / 高清
- 命名：`原文件名_模板名_模特`
- 导出：单张下载 / 批量 ZIP

---

## 6. 非功能要求（决定是否“能卖”）

- 单图平均生成时长：<= 18 秒（目标）
- 首屏可预览时间：<= 25 秒
- 批量任务成功率：>= 90%
- 同批一致性：同模板风格一致、色调稳定
- 异常可恢复：失败项可单独重试

---

## 7. 成功指标（北极星与辅助）

### 7.1 北极星指标
- 每周至少使用 2 次模特图功能的商家占比

### 7.2 功能指标
- 上传 -> 生成 -> 导出 完成率
- 首次可用率（无需二修）
- 单图平均耗时
- 质检警告率

### 7.3 商业指标
- 白底用户升级使用模特图比例
- 模特图功能 7 日留存
- 付费商家月均导出张数

---

## 8. 定价与打包建议（验证期）

- 试用包：首周免费 20 张模特图
- 联动包：白底 + 模特组合包（优先推荐）
- 增量包：按量购买（100 张/包）

策略原则：
- 先卖“效率闭环”而不是卖单点 AI 能力
- 强化“今天就能出素材，明天就能卖货”

---

## 9. 与“端到端效率优势”相关策略

1. 白底图结果页一键流转，减少重复上传
2. 模板化优先，降低学习成本
3. 批量与一致性优先，服务真实商家工作流
4. 与海报页打通，形成“商品图 -> 场景图 -> 宣发图”闭环

---

## 10. 风险与兜底

- 风险1：模特肢体异常导致可用率低
  - 兜底：异常检测 + 一键重试 + 推荐模板替代
- 风险2：风格过重遮挡服装卖点
  - 兜底：默认中性背景强度 + 商品细节优先模式
- 风险3：商家认为“可看但不够稳定”
  - 兜底：模板白名单 + 商家私有常用模板

---

## 11. V1 里程碑建议

- D1-D3：打通上传-模板-生成-导出最小流程
- D4-D7：补齐批量与基础质检
- D8-D10：接入白底图一键流转 + 导出命名
- D11-D14：真实商家试用与参数收敛
